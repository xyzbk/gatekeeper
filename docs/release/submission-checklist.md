# Submission checklist

## Verified local release evidence

- [ ] Run the full Phase 7 quality gate and record exact output in the execution log.
- [ ] Run the network-free judge demo and golden evaluation report.
- [ ] Review the final README, security statement, screenshots, and demo narration against the product actually implemented.

## Requires user authorization

- [ ] Capture the requested Codex `/feedback` session identifier from the session selected by the user; do not invent or publish one.
- [ ] Upload the prepared public YouTube video after the user approves its final cut.
- [ ] If the repository is private, grant the required judge accounts access after the user approves the target repository and access scope.
- [ ] Create/finalize the Devpost project in the Developer Tools category after the user approves the final copy and assets.
- [ ] Submit Devpost only after the user explicitly instructs submission.

## Code-freeze rule

After Phase 7 acceptance, only release-blocking bug fixes, documentation corrections, video edits, and submission validation are allowed. New product features remain out of scope.
